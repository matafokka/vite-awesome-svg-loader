# Code of conduct

## Purpose

This document:

1. Establishes community guidelines.
2. ~~Farms aura~~ Improves package's discoverability.

## Community guidelines

**Guidelines:**

1. Comply with [GitHub's community guidelines](https://docs.github.com/en/site-policy/github-terms/github-community-guidelines).
1. Be nice.

Examples of **good** behavior:

1. Being (generally, if possible) a nice person.
2. Sticking to the discussion in context of this repository only.

Examples of **bad** behavior:

1. Being a meanie.
2. Discussing things that are disconnected from this repository.

## Scope

This repository.

## Enforcement

Warning after first violation. Ban after second violation. Immediate ban after the first serious violation like a death threat.

## Reasoning

Publishing platforms increase package's scoring when code of conduct is present.

This is a minimal code of conduct that is based on common sense.

This document is, subjectively, less ambiguous than existing code of conducts. This is an improvement considering that the whole thing is a joke.

Why is this a joke? Because the community behavior is already governed by GitHub. Why would one expect some guy who builds a project in his free time to be a whole police force?
